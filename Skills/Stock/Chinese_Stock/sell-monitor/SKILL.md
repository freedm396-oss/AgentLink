# 持仓监控 SKILL（v4 简化版）

> 每日 09:20 和 14:30 运行，基于持仓与推荐列表给出交易建议。

---

## 核心逻辑

```
持仓列表 ──┬── 在推荐列表中 ──► 加仓建议（未触发止盈时）
           │
           └── 不在推荐列表中 ──► 检查止盈/止损 ──► 减仓建议（触发时）
                                                      │
                                                      └── 持有建议（未触发时）

推荐列表 ──┬── 在持仓列表中 ──► 已处理（见上）
           │
           └── 不在持仓列表中 ──► 建仓建议
```

### 三种交易信号

| 信号 | 条件 | 操作 |
|------|------|------|
| **建仓** | 在推荐列表，不在持仓中 | 买入 |
| **加仓** | 同时在持仓和推荐列表，且未触发止盈 | 增持 |
| **减仓** | 在持仓中，触发止盈或止损条件 | 减持/清仓 |

---

## 止盈止损规则

### 1. 固定比例止盈止损

基于 `buy_reason`（买入原因）的差异化参数：

| buy_reason | 止损线 | 止盈阶梯 |
|-----------|--------|---------|
| `limit_up` / `breakout_high` | **-7%** | 15%→减20%, 30%→减30%, 40%→减50%, 50%→减70%, 80%→清仓 |
| `gap_fill` / `ma_bullish` | **-5%** | 10%→减30%, 15%→减50%, 20%→减50%, 30%→减70%, 40%→清仓 |
| `macd_divergence` / `rsi_oversold` / `morning_star` / `volume_extreme` | **-3%** | 5%→减20%, 8%→减30%, 10%→减50%, 15%→减70%, 20%→清仓 |
| `a_stock_1430` | **-2%** | 2%→减50%, 5%→清仓 |

### 2. 动态回撤止盈

当股票从买入后最高价回撤一定比例时触发：

| 策略类型 | 回撤阈值 | 触发条件 |
|---------|---------|---------|
| 激进 (`limit_up`/`breakout_high`) | **10%** | 从高点回撤10%，且曾盈利≥5% |
| 稳健 (`gap_fill`/`ma_bullish`) | **8%** | 从高点回撤8%，且曾盈利≥5% |
| 保守 (其他技术策略) | **5%** | 从高点回撤5%，且曾盈利≥5% |
| 超短 (`a_stock_1430`) | **3%** | 从高点回撤3%，且曾盈利≥5% |

> **示例**：买入后涨到+50%（高点），随后回撤到+35%（回撤15%），触发激进策略的10%回撤止盈，建议减仓50%保护利润。

### 优先级
**止损 > 固定比例止盈 > 动态回撤止盈 > 加仓 > 持有**

## 依赖数据

### 持仓文件
- 路径：`Chinese_Stock/my_holdings/holdings.json`
- 或：`Chinese_Stock/my_holdings/YYYYMMDD_holdings.json`（自动读取最新）

格式：
```json
[
  {
    "code": "600105",
    "name": "永鼎股份",
    "buy_price": 38.10,
    "shares": 1000,
    "buy_reason": "breakout_high",
    "buy_date": "2026-04-11"
  }
]
```

### 推荐文件
- 路径：`Chinese_Stock/recommendations/YYYYMMDD_*_recommendation.json`
- 自动读取最新日期的推荐文件

格式：
```json
[
  {
    "code": "600105",
    "name": "永鼎股份",
    "source": "breakout_high",
    "reason": "突破新高，趋势确认",
    "entry_price": 38.50
  }
]
```

## 运行方式

### 手动运行

```bash
cd ~/.npm-global/lib/node_modules/openclaw/skills/Stock/Chinese_Stock/sell-monitor/scripts
python3 monitor.py
```

### 定时运行（cron）

```yaml
# crons/sell-monitor-crons.yaml
jobs:
  - name: "sell-monitor-morning"
    schedule: "0 20 9 * * 1-5"  # 工作日 09:20
    command: "python3 scripts/monitor.py"
    
  - name: "sell-monitor-afternoon"
    schedule: "0 30 14 * * 1-5"  # 工作日 14:30
    command: "python3 scripts/monitor.py"
```
